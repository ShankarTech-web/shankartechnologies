1. The 5 Groups of SQL Queries Core Concept

All SQL Queries utilize commands that are divided into 5 distinct groups[cite: 1, 127]:

1. Data Definition Language (DDL)

Used to define and modify the structure (schema) of database objects[cite: 1, 35].

  • CREATE: Create DB or Table.
  • ALTER: Add/Drop columns.
  • DROP: Delete table structure + data.
  • TRUNCATE: Remove all records, keep structure.
2. Data Manipulation Language (DML)

Used to manage and modify the data within the schema[cite: 1, 41].

  • INSERT: Add new rows.
  • UPDATE: Modify existing rows.
  • DELETE: Remove specific rows.
3. Data Query Language (DQL)

Used solely to retrieve data[cite: 1, 47].

  • SELECT: The primary command to fetch data.
4. DCL & 5. TCL

DCL (Control): Permissions[cite: 1].
TCL (Transaction): Data Integrity[cite: 1].

  • GRANT / REVOKE (DCL)
  • COMMIT / ROLLBACK (TCL)

2. SQL Constraints

Constraints are rules applied to columns to ensure data integrity[cite: 253].

Constraint Description
NOT NULL Ensures a column cannot accept null values[cite: 254].
UNIQUE Ensures all values in a column are different[cite: 254].
PRIMARY KEY Combination of NOT NULL + UNIQUE[cite: 254].
FOREIGN KEY A reference to a Primary Key in another table[cite: 254].
CHECK Ensures values satisfy a specific condition (e.g., age > 18)[cite: 254].
DEFAULT Sets a default value if none is specified[cite: 254].
IDENTITY Auto-increments the column value automatically[cite: 254].
Example Syntax
CREATE TABLE employee (
    id INT PRIMARY KEY IDENTITY(101, 1), 
    firstname VARCHAR(20) NOT NULL, 
    age INT CHECK(age > 18),
    gender VARCHAR(30) DEFAULT 'male'
 ); [cite: 257]

3. Clauses & Operators

GROUP BY

Groups rows that have the same values (often used with Aggregates like COUNT, SUM)[cite: 58, 59].

SELECT circle, COUNT(offers) 
FROM endcustomer 
 GROUP BY circle; [cite: 61]

ORDER BY

Sorts the result-set in Ascending (ASC) or Descending (DESC) order[cite: 260].

SELECT * FROM employee 
 ORDER BY salary DESC; [cite: 261]
LIKE Operator: Used for pattern matching[cite: 264].
  • % : Matches multiple characters.
  • _ : Matches exactly one character.

4. SQL Joins Important

Joins are used to combine rows from two or more tables[cite: 203].

Inner Join

Returns records that have matching values in both tables[cite: 203].
Left Join

Returns all records from the left table, and matched records from the right[cite: 208].
Right Join

Returns all records from the right table, and matched records from the left[cite: 211].
Full Join

Returns all records when there is a match in either left or right table[cite: 214].

5. Advanced Concepts

Saved SQL code that can be reused. It can accept parameters[cite: 94].

CREATE PROCEDURE updateSalaryById
    @SalaryValue INT, @IdValue INT
AS BEGIN
    UPDATE employee SET salary = @SalaryValue 
    WHERE id = @IdValue;
 END; [cite: 100]

Used to find Nth highest/lowest salaries. DENSE_RANK handles duplicates without skipping numbers[cite: 85].

-- Find 2nd lowest salary using CTE (WITH clause)
WITH tempEmployee AS (
    SELECT *, DENSE_RANK() OVER (ORDER BY salary ASC) AS rnk 
    FROM employee
)
 SELECT * FROM tempEmployee WHERE rnk = 2; [cite: 91]

Understanding the difference is vital for interviews.

Feature DELETE [cite: 24] TRUNCATE [cite: 29] DROP [cite: 26]
Group DML DDL DDL
Action Deletes specific or all rows. Removes ALL rows (reset). Removes Table + Data.
Speed Slowest. Faster than Delete. Fastest.
Rollback Yes[cite: 25]. No (generally)[cite: 29]. No[cite: 27].
