Comprehensive SQL Guide

This document covers the 5 sub-languages of SQL, constraints, joins, clauses, and advanced concepts like Stored Procedures.

1. The 5 Groups of SQL Queries

All SQL Queries utilize commands that are divided into 5 distinct groups[cite: 1, 156]:

1. Data Definition Language (DDL)

Used to define and modify the structure (schema) of database objects.

  • CREATE: Creates a new table or database[cite: 58].
  • ALTER: Adds, removes, or renames columns[cite: 31].
  • DROP: Deletes an entire table or database (structure + data)[cite: 16].
  • TRUNCATE: Removes all records but keeps the structure[cite: 20].
-- Example: Adding a column
ALTER TABLE employee ADD country VARCHAR(20); [cite: 32]

2. Data Manipulation Language (DML)

Used to manage and modify the data within the schema[cite: 30].

  • INSERT: Adds new records. Can insert single or multiple rows[cite: 156].
  • UPDATE: Modifies existing data. Can use `CASE` for conditional updates[cite: 45].
  • DELETE: Removes specific or all records[cite: 5].
-- Example: Update with CASE
UPDATE register SET lastname = CASE id 
    WHEN 1 THEN 'abc'
    WHEN 2 THEN 'pqr'
    ELSE lastname END
WHERE id IN (1,2); [cite: 45]

3. Data Query Language (DQL)

Used solely to retrieve data[cite: 61].

  • SELECT: Retrieves data from a database table.

4. Data Control Language (DCL) & 5. Transaction Control Language (TCL)

Manage permissions (GRANT, REVOKE) and transactions (COMMIT, ROLLBACK, SAVEPOINT)[cite: 1].

2. SQL Constraints

Constraints are rules applied to columns to ensure data integrity[cite: 67].

Constraint Definition
NOT NULL Ensures a column cannot accept null values[cite: 68].
UNIQUE Ensures all values in a column are different[cite: 68].
PRIMARY KEY Combination of `NOT NULL` + `UNIQUE`.Uniquely identifies each record[cite: 68].
FOREIGN KEY A field that refers to the Primary Key in another table[cite: 68].
CHECK Ensures that the values in a column satisfy a specific condition (e.g., age > 18)[cite: 68, 69].
DEFAULT Sets a default value for a column if no value is specified[cite: 68].
IDENTITY Auto-increments the column value automatically (e.g., starting at 101, incrementing by 1)[cite: 68, 71]. 
-- Example Table Creation
CREATE TABLE employee (
    id INT PRIMARY KEY IDENTITY(101, 1), 
    firstname VARCHAR(20) NOT NULL, 
    age INT CHECK(age > 18),
    gender VARCHAR(30) DEFAULT 'male'
); [cite: 71]

3. Clauses and Operators

ORDER BY Clause

Used to sort the result-set in ascending (ASC) or descending (DESC) order[cite: 74].

SELECT * FROM employee ORDER BY salary DESC; [cite: 75]

LIKE Operator (Pattern Matching)

Used in a WHERE clause to search for a specified pattern in a column.

  • % (Percent): Represents zero, one, or multiple characters[cite: 78].
  • _ (Underscore): Represents exactly one single character[cite: 78].
-- Finds firstname starting with any character, but second character is 'a'
SELECT * FROM employee WHERE firstname LIKE '_a%'; [cite: 79]

GROUP BY Clause

Groups rows that have the same values into summary rows. It is often used with aggregate functions like COUNT, MAX, MIN, SUM, AVG[cite: 179, 180].

-- Example: Count offers per circle
SELECT circle, COUNT(offers) FROM endcustomer GROUP BY circle; [cite: 182]

4. SQL Joins

A JOIN clause is used to combine rows from two or more tables, based on a related column between them. There are 6 main types[cite: 248].

Types of Joins:

  • Inner Join: Returns records that have matching values in both tables[cite: 248].
  • Left Join: Returns all records from the left table, and the matched records from the right table. (Returns NULL for no match) [cite: 253].
  • Right Join: Returns all records from the right table, and the matched records from the left table[cite: 256].
  • Full Join: Returns all records when there is a match in either left or right table[cite: 260].
  • Cross Join: Returns the Cartesian product (all combinations) of the two tables[cite: 263].
  • Self Join: A regular join, but the table is joined with itself[cite: 264].
-- Example: Joining 3 Tables (Inner Join)
SELECT * FROM employee 
INNER JOIN dept ON employee.id = dept.id
 INNER JOIN register ON employee.id = register.id; [cite: 103]

5. Advanced Concepts

Stored Procedures

A stored procedure is a prepared SQL code that you can save, so the code can be reused over and over again[cite: 215]. It can accept parameters.

-- Creating a procedure to update salary
CREATE PROCEDURE updateSalaryById
    @SalaryValue INT, @IdValue INT
AS BEGIN
    UPDATE employee SET salary = @SalaryValue WHERE id = @IdValue;
 END; [cite: 221]

-- Executing the procedure
 EXEC updateSalaryById @SalaryValue = 999, @IdValue = 5; [cite: 222]

Ranking Functions (DENSE_RANK)

DENSE_RANK() assigns a rank to each row within a partition of a result set. It handles duplicates by giving them the same rank and does not skip the next rank number[cite: 206, 207].

-- Finding the 2nd lowest salary using a Subquery
SELECT * FROM (
    SELECT *, DENSE_RANK() OVER (ORDER BY salary ASC) AS rnk FROM employee
 ) AS tempEmployee WHERE rnk = 2; [cite: 211]

Common Table Expressions (CTE / WITH Clause)

The WITH clause defines a temporary result set (CTE) that you can reference within another SELECT, INSERT, UPDATE, or DELETE statement[cite: 208].

-- Using WITH to find Nth highest salary
WITH tempEmployee AS (
    SELECT *, DENSE_RANK() OVER (ORDER BY salary DESC) AS rnk FROM employee
)
 SELECT * FROM tempEmployee WHERE rnk = 5; [cite: 210]

6. Critical Interview Question: DELETE vs DROP vs TRUNCATE

This is a very important concept[cite: 24, 30].

Feature DELETE [cite: 24, 25, 26] TRUNCATE [cite: 21, 22, 29] DROP [cite: 16, 17, 18, 28]
Group DML (Data Manipulation) DDL (Data Definition) DDL (Data Definition)
What it does Deletes specific rows (based on WHERE) or all rows. Removes ALL rows from the table. Deletes the entire table structure and data.
Structure Table structure remains. Table structure remains. Table structure is removed.
