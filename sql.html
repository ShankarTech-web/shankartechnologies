Introduction to SQL

SQL is a standard language for accessing and manipulating databases.

What is SQL?

  • SQL stands for Structured Query Language.
  • SQL lets you access and manipulate databases.
  • SQL became a standard of the American National Standards Institute (ANSI) in 1986, and of the International Organization for Standardization (ISO) in 1987.

What Can SQL Do?

  • Execute queries against a database
  • Retrieve data from a database
  • Insert records in a database
  • Update records in a database
  • Delete records from a database
  • Create new databases
  • Create new tables in a database
  • Create stored procedures in a database
  • Create views in a database
  • Set permissions on tables, procedures, and views

The 5 Groups of SQL Queries

All SQL Queries utilize commands that are divided into 5 distinct groups:

  1. Data Definition Language (DDL)
  2. Data Manipulation Language (DML)
  3. Data Query Language (DQL)
  4. Data Control Language (DCL)
  5. Transaction Control Language (TCL)

1. Data Definition Language (DDL)

Purpose: Used to define and modify the structure (schema) of the database objects (tables, indexes, views), not the data inside them.

CREATE
Creates a new table, database, or other objects.
ALTER
Modifies an existing database object (e.g., adding a column).
DROP
Deletes an entire table or database object (structure + data).
TRUNCATE
Removes all records from a table but keeps the structure.
RENAME
Renames an object.
Note: DDL commands are usually "auto-committed," meaning the changes are saved permanently immediately.

2. Data Manipulation Language (DML)

Purpose: Used to manage and modify the data present within the schema.

INSERT
Inserts new data (rows) into a table.
UPDATE
Updates existing data within a table.
DELETE
Deletes specific records (rows) from a table.

3. Data Query Language (DQL)

Purpose: Used solely to retrieve and fetch data from the database. It does not modify data or structure.

SELECT
The primary command used to query data.
Note: In some contexts, DQL is grouped under DML, but keeping it separate is accurate because SELECT is non-destructive (read-only).

4. Data Control Language (DCL)

Purpose: Used to handle rights, permissions, and security of the database system.

GRANT
Gives users access privileges to the database.
REVOKE
Withdraws user access privileges.

5. Transaction Control Language (TCL)

Purpose: Used to manage transactions within the database. These commands play a vital role in maintaining data integrity, usually working alongside DML commands.

COMMIT
Saves the work done in a transaction permanently.
ROLLBACK
Restores the database to the state it was in before the last commit (undoes changes).
SAVEPOINT
Sets a point within a transaction to which you can later roll back.
